# GitHub Projects Profile Display Design

## Goal

Let a user enter a GitHub profile URL, fetch all public repositories for that user, generate ATS-friendly project bullets with the configured LLM API, and display the extracted repository details in the CareerOS profile UI.

## Scope

This version is for checking the end-to-end flow, so it displays all public repositories returned by GitHub. Filtering by forks, pinned repositories, stars, recency, or project relevance is out of scope for now.

## User Flow

1. The user enters a GitHub profile URL in the existing GitHub Profile field.
2. The user clicks a GitHub projects action in the profile form.
3. CareerOS calls `/api/projects-summary` with the GitHub URL.
4. The backend extracts the username, fetches all public repositories, asks OpenRouter for ATS-friendly bullet points, and returns structured project summaries.
5. The frontend displays a GitHub Projects section with one card per repository.

## Backend

Update `POST /api/projects-summary` to accept either:

```json
{ "username": "octocat" }
```

or:

```json
{ "githubUrl": "https://github.com/octocat" }
```

The route will validate the input, fetch all public repos using `fetchGitHubUserProjects`, pass repo metadata to `generateProjectSummaries`, and return:

```json
{
  "projects": [
    {
      "title": "repo-name",
      "url": "https://github.com/octocat/repo-name",
      "description": "Repository description",
      "language": "TypeScript",
      "topics": ["nextjs"],
      "stars": 10,
      "atsPoints": ["Built ...", "Implemented ..."]
    }
  ]
}
```

## Frontend

The profile form will keep GitHub project summaries in component state separately from the existing profile result. This avoids forcing GitHub-derived summaries into the resume/profile schema before the data shape is stable.

The GitHub Projects section will display:

- Repository title.
- GitHub link.
- Language, topics, and star count when available.
- ATS-friendly bullet points generated by the LLM.

## Errors

Missing GitHub URL returns a local validation message.

Invalid GitHub profile input returns `400`.

GitHub user not found returns `404`.

OpenRouter configuration or generation failures return a visible error message.

Empty public repositories return an empty GitHub Projects state.

## Testing

Add route tests for username input, GitHub URL input, missing input, invalid GitHub URL, GitHub errors, and OpenRouter errors.

Add frontend tests for rendering the GitHub project button, calling the API with the entered GitHub URL, rendering project cards, and showing validation errors.
